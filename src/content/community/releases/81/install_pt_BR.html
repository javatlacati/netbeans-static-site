Instruções de Instalação do NetBeans IDE 8.1

Última atualização: 26 de outubro de 2015

Este documento descreve como instalar o NetBeans IDE 8.1 no seu sistema. Consulte as Notas da Release do NetBeans IDE 8.1 para obter informações sobre os sistemas operacionais e configurações de hardware compatíveis para o IDE. Para saber mais sobre as novas funcionalidades incluídas nesta release do IDE, consulte a página Informações da Release do NetBeans IDE 8.1.

Software Necessário

É necessário ter o Java SE Development Kit (JDK) 7, Atualização 10 (ou posterior), ou o JDK 8, para instalar o NetBeans IDE. Você pode fazer download da última atualização do JDK 7 e do JDK 8 em http://www.oracle.com/technetwork/java/javase/downloads.

Os bundles PHP e C/C++ NetBeans requerem apenas que o JRE (Java Runtime Environment) 7 ou 8 seja instalado e esteja em execução. O JDK 7 ou 8 é obrigatório se você pretende usar qualquer uma das funcionalidades Java.

O JDK 7 Update 6 (ou mais recente) é obrigatório para usar as funcionalidades do JavaFX 2.2 (ou mais recente) no NetBeans IDE 8.1.

O JDK 8 é obrigatório para usar as funcionalidades do JavaFX 8 no NetBeans IDE 8.1.

Observação:

  • O JavaFX 2.2 está no mesmo pacote que o mais recente JDK 7 para Windows, Mac e Linux.
  • O JavaFX 8 está no mesmo bundle que o JDK 8 para Windows, Mac e Linux.
  • Diferente das versões anteriores do NetBeans IDE, o NetBeans IDE 8.1 não requer que você configure uma plataforma Java ativada para FX para utilizar o suporte e JavaFX no IDE. Você poderá desenvolver projetos JavaFX no IDE se instalar qualquer plataforma Java padrão, isto é, JDK 7 Update 6 ou mais recente, ou JDK 8.
  • Para usar as funcionalidades do JavaFX no NetBeans IDE, recomendamos enfaticamente que você faça o upgrade de sua versão do JDK para o JDK 7 Update 10 (ou versão mais recente) ou então que use o JDK 8.

Opções de Download do Instalador

Na página Download do NetBeans IDE, você pode adquirir um dos diversos instaladores, cada um contendo o IDE básico e as ferramentas adicionais.

Os seguintes instaladores estão disponíveis:

  • Java SE. Suporta todas as funcionalidades de desenvolvimento padrão do Java SE e também para desenvolvimento da Plataforma NetBeans e para o JavaFX 2.2 SDK (ou mais recente) ou JavaFX 8 SDK.
  • Java EE. Fornece ferramentas para o desenvolvimento de aplicações Java SE e Java EE, assim como suporte para o desenvolvimento da Plataforma NetBeans e para o JavaFX 2.2 SDK (ou mais recente), ou JavaFX 8 SDK. Esta opção de download inclui também o GlassFish Server Open Source Edition 4.1.1 e o software Apache Tomcat 8.0.27.
  • HTML5/JavaScript. Fornece ferramentas para desenvolvimento HTML5/Javascript. Inclui o Java Runtime Environment e não exige uma instalação Java distinta.
  • PHP. Fornece ferramentas para o desenvolvimento PHP 5.x e o suporte para o Zend e Symfony Framework. Inclui o Java Runtime Environment e não exige uma instalação Java distinta.
  • C/C++. Oferece suporte para desenvolvimento nas linguagens C, C++, Fortran e Assembly. Inclui o Java Runtime Environment e não exige uma instalação Java distinta.
  • Tudo. Esta é uma opção de download completa, que contém todos os runtimes e tecnologias disponíveis para o NetBeans IDE.

Personalizando Sua Instalação

Se você tiver feito download da opção Tudo, poderá escolher exatamente as ferramentas e os runtimes a serem instalados. Clique em Personalizar na página de Boas-Vindas do instalador e selecione e as funcionalidades que deseja utilizar. Para obter mais informações, consulte Instalando o Software.

Se você deseja adicionar funcionalidade a sua configuração do NetBeans IDE, utilize o Gerenciador de Plug-ins do NetBeans. Por exemplo, presuma que você tenha iniciado pelo download e instalação do pacote do PHP. Em seguida, você pode decidir se deseja experimentar a funcionalidade C/C++. Para fazer isso, acesse o Gerenciador de Plug-ins do NetBeans no IDE (selecione Ferramentas > Plug-ins) e adicione o pacote C/C++ à instalação existente.

Há diversas observações importantes sobre a instalação e a configuração de alguns recursos do NetBeans IDE:

  • O GlassFish Server Open Source Edition 4.1.1 está incluído nas opções de download Java EE e Tudo, mas não está instalado por default nessas opções. Para instalar o GlassFish Server Open Source Edition 4.1.1, selecione a opção adequada abaixo:
    • no download do Java EE, inicie o instalador e marque a caixa de seleção GlassFish Server Open Source Edition 4.1.1 na página de Boas-Vindas;
    • no download de Tudo, inicie o instalador e selecione GlassFish Server Open Source Edition 4.1.1 na caixa de diálogo Personalizar Instalação.
    Você deve alterar a senha default de administrador após instalar o GlassFish Server. Consulte FaqSettingGlassFishPassword para obter detalhes.
  • O Apache Tomcat 8.0.27 está incluído nas opções de download Java EE e Tudo, mas não está instalado por default nessas opções. Para instalar o Apache Tomcat, selecione a opção adequada abaixo:
    • no download do Java EE, inicie o instalador e marque a caixa de seleção Apache Tomcat 8.0.27 na página de Boas-Vindas;
    • no download de Tudo, inicie o instalador e selecione Apache Tomcat 8.0.27 na caixa de diálogo Personalizar Instalação.
  • Para desenvolvimento C/C++ (aplicável para os downloads C/C++ e Tudo), instale também os compiladores e as ferramentas. Para obter mais informações, consulte o documento Configuring the NetBeans IDE for C/C++/Fortran. <-->
  • Para desenvolvimento de HTML5 e JavaScript para dispositivos móveis (Android e iOS), você precisa adicionar SDKs e outras ferramentas de desenvolvimento à plataforma de desenvolvimento do alvo. Para obter mais informações, consulte a página wiki Browsers Móveis.
  • Para OS X, você pode personalizar a instalação do NetBeans IDE dos downloads Tudo e Java EE. A instalação do IDE dos downloads Java SE, HTML5/JavaScript, PHP e C/C++ não pode ser personalizada.
  • Também é possível fazer download de um arquivo zip independente de plataforma e executar o arquivo executável do NetBeans IDE. Observe que o GlassFish Server Open Source e o Apache Tomcat estão disponíveis somente com instaladores específicos para cada plataforma.
  • Para tornar modelos de projeto de amostra inacessíveis na instalação do NetBeans IDE, consulte a página FAQ Ocultar Projetos de Amostra.

Iniciando o Download

  1. Acesse https://netbeans.org/downloads/.
  2. Na parte superior direita da página, selecione a linguagem e a plataforma na lista drop-down. Também é possível optar por fazer download e utilizar somente o arquivo zip independente da plataforma.
  3. Clique no botão Download para selecionar a opção de download que deseja instalar.
  4. Salve o arquivo do instalador no sistema.

Instalando o Software

Siga as instruções nesta seção para instalar o IDE no sistema. Estas instruções de instalação se aplicam a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis e dos requisitos do sistema, consulte as notas da release.

Microsoft Windows e Linux

Para instalar o software:

  1. Após concluir o download, execute o instalador.
    • Para Windows, o arquivo executável do instalador tem a extensão .exe. Clique duas vezes no arquivo do instalador para executá-lo.
    • Nas plataformas Linux, o arquivo do instalador tem a extensão .sh. Nessas plataformas, é preciso criar os arquivos executáveis do instalador usando este comando: chmod +x <installer-file-name>. Digite ./<installer-file-name> para executar o instalador.
  2. Se você tiver feito download do pacote Tudo ou Java EE, poderá personalizar a sua instalação. Execute as seguintes etapas na página de Boas-Vindas do assistente de instalação:
    1. Clique em Personalizar.
    2. Na caixa de diálogo Personalizar Instalação, faça suas seleções.
    3. Clique em OK.
  3. Clique em Próximo na página de Boas-Vindas do assistente de instalação.
  4. Na página Contrato de Licença, leia o contrato de licença, clique na caixa de seleção para aceitar e clique em Próximo.
  5. Na página de instalação do IDE NetBeans, proceda da seguinte maneira:
    1. Aceite o diretório default de instalação do IDE NetBeans ou especifique outro diretório.

      Observação: o diretório de instalação deverá estar vazio e o perfil de usuário que estiver sendo utilizado para executar o instalador deverá ter permissões de leitura/gravação nesse diretório.

    2. (Aplicável somente ao pacote Tudo ou Java EE.) Aceite a instalação default do JDK para utilizar com o NetBeans IDE ou selecione outra instalação na lista drop-down. Se o assistente de instalação não tiver encontrado uma instalação JDK compatível para ser utilizada com o NetBeans IDE, é porque o JDK não é instalado no local default. Nesse caso, especifique o caminho do JDK instalado e clique em Próximo ou cancele a instalação atual. Após a instalação da versão necessária do JDK, você poderá reiniciar a instalação.

      Observação: se a versão do JDK for mais antiga que a JDK 7 Update 10 recomendada, faça o download e instale a atualização mais recente do JDK pela páginaDownloads do Java SE e reinicie o instalador do NetBeans IDE.

  6. Se a página de instalação do GlassFish Server Open Source Edition 4.1.1 for aberta, aceite o diretório de instalação default ou especifique outro local de instalação.
  7. Se estiver instalando o Apache Tomcat, em sua página de instalação, aceite o diretório de instalação default ou especifique outro local de instalação. Clique em Próximo.
  8. Na página Resumo, faça o seguinte:
    1. Verifique se a lista de componentes a serem instalados está correta.
    2. Marque a caixa de seleção Verificar Atualizações se você quiser verificar possíveis atualizações na Central de Atualização.
    3. Verifique se há espaço suficiente no sistema para a instalação.
  9. Clique em Instalar para iniciar a instalação.
  10. Na página Configuração Concluída, forneça os dados de utilização anônimos, se desejado, e clique em Finalizar.

Observação: se você tiver problemas após concluir instalação do software com sucesso, consulte Diagnóstico e Solução de Problemas para obter descrições e soluções para questões não-resolvidas que poderão afetar o processo de instalação.

OS X

Para instalar o software:

  1. Após concluir o download, execute o instalador. O arquivo do instalador tem a extensão .dmg.
  2. Clique no ícone de pacote no painel exibido. A extensão do pacote é .pkg . O assistente de instalação é iniciado.
  3. Clique em Continuar quando for exibida a caixa de diálogo "Este pacote executará um programa para determinar se o software pode ser instalado".
  4. Na página de Introdução do assistente de instalação, clique em Continuar.

    Observação: se a versão do JDK for mais antiga que a JDK 7 Update 10 recomendada, faça o download e instale a atualização mais recente do JDK pela páginaDownloads do Java SE e reinicie o instalador do NetBeans IDE.

  5. Leia o contrato de licença e clique em Continuar. Clique em Aceitar na janela pop-up para aceitar a licença.
  6. Na página Selecionar um Destino, selecione o drive e clique em Continuar.
  7. Se você tiver feito download do pacote Tudo ou Java EE, poderá personalizar a sua instalação. Pressione o botão Personalizar no lado esquerdo inferior do painel do assistente de instalação, no último painel do assistente de instalação. A árvore de produtos é exibida.
  8. Selecione os produtos que deseja instalar.
  9. Para iniciar a instalação, digite o nome do administrador e a senha no sistema e clique em OK.

Pacote Independente de Plataforma

Para executar o software:

  1. Após fazer download do arquivo ZIP independente de plataforma, extraia-o em qualquer pasta do sistema.
  2. Execute o arquivo executável localizado no diretório netbeans/bin.
  3. Aceite o Contrato de Licença.
  4. Clique em Próximo. A instalação do NetBeans IDE se inicia.

Observação: se não for possível encontrar uma instalação do JDK compatível, talvez seja necessário adicionar manualmente o caminho ao diretório de instalação do JDK, adotando o seguinte procedimento:

  1. Abra o arquivo netbeans.conf localizado no diretório netbeans/etc em um editor de texto.
  2. Insira a localização da instalação do JDK compatível para a opção netbeans_jdkhome. O local default no Windows é C:\Program Files\Java\jdk1.7.0_67 ou similar.
  3. Salve o arquivo netbeans.conf e execute o arquivo executável no diretório netbeans/bin.

Observação: o GlassFish Server Open Source Edition e o Apache Tomcat somente estão disponíveis com instaladores específicos para cada plataforma.

Observação: para usar o GlassFish Server Open Source Edition e/ou Apache Tomcat já instalados na máquina, no NetBeans IDE, clique em Servidores no menu Ferramentas. Dessa forma, o Assistente de Adição de Instância de Servidor será exibido, o que vai ajudá-lo a registrar uma instância do servidor Web necessário com o NetBeans IDE.

Desinstalando o Software

Siga as instruções nesta seção para desinstalar o IDE do seu sistema. Estas instruções de desinstalação se aplicam a todas as plataformas compatíveis. Para obter uma lista das plataformas compatíveis, consulte as notas da release.

Microsoft Windows

Se necessário, é possível desinstalar o NetBeans IDE, o GlassFish Server Open Source 4.1.1 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o IDE:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas, selecione NetBeans IDE 8.1 e clique em Remover.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo, assegure-se de que as versões listadas são as corretas e de que as caixas apropriadas estão marcadas.
  4. Marque a caixa de seleção de diretório de usuário do NetBeans se quiser removê-lo com o IDE e clique em Desinstalar.
  5. Na página Resumo, clique em Desinstalar.
  6. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o GlassFish Server Open Source Edition 4.1.1:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas. Em seguida, selecione GlassFish Server Open Source Edition 4.1.1 e clique em Remover.

Para desinstalar o Apache Tomcat:

  1. Faça shutdown do IDE.
  2. No Painel de Controle, selecione Adicionar ou Remover Programas. Em seguida, selecione Apache Tomcat 8.0.27 e clique em Remover.

Linux

Se necessário, é possível desinstalar o NetBeans IDE, o GlassFish Server Open Source 4.1.1 e o Apache Tomcat utilizando os procedimentos a seguir.

Para desinstalar o IDE:

  1. Faça shutdown do IDE.
  2. No diretório de instalação do IDE, execute o arquivo uninstall.sh.
  3. Para desinstalar o GlassFish, o Tomcat e o IDE ao mesmo tempo, na página Resumo, assegure-se de que as versões listadas são as corretas e de que as caixas apropriadas estão marcadas.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o GlassFish Server Open Source Edition 4.1.1:

  1. Faça shutdown do IDE.
  2. Localize o diretório de instalação do servidor de aplicações.

    Observação: o local de instalação default será diferente, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório de instalação default do servidor de aplicações para várias plataformas.

    Plataforma Instalando Como... Instalação do GlassFish 4.1.1
    Windows usuário com direitos de administrador %ProgramFiles%\glassfish-4.1.1
    Windows usuário %USERPROFILE%\glassfish-4.1.1
    Linux usuário ~/glassfish-4.1.1
    Linux raiz /usr/local/glassfish-4.1.1
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar.

Para desinstalar o Apache Tomcat:

  1. Faça shutdown do IDE.
  2. Localize o diretório de instalação do Apache Tomcat.

    Observação: o local de instalação default varia, dependendo do tipo de instalação e da plataforma. A tabela a seguir exibe o diretório de instalação default do servidor de aplicações para diferentes plataformas.

    Plataforma Instalando Como... Instalação do Apache Tomcat 8.0.27
    Windows usuário com direitos de administrador %ProgramFiles%\Apache Software Foundation\Apache Tomcat 8.0.27
    Windows usuário %USERPROFILE%\Apache Software Foundation\Apache Tomcat 8.0.27
    Linux usuário ~/apache-tomcat-8.0.27
    Linux raiz /usr/local/apache-tomcat-8.0.27
  3. No diretório de instalação do servidor de aplicações, execute o arquivo uninstall.sh.
  4. Na página Resumo, clique em Desinstalar.
  5. Quando a desinstalação estiver concluída, clique em Finalizar

OS X

Para desinstalar o IDE e o servidor de aplicações:

  1. Faça shutdown do software.
  2. Arraste as pastas ou arquivos de programas para a Lixeira. Observe que o local default do IDE e do servidor de aplicações é /Applications/NetBeans/.

Pacote Independente de Plataforma

Para desinstalar os arquivos do IDE, delete os diretórios de instalação do IDE.

Upgrade e Suporte a Várias Instalações

Várias instalações do NetBeans IDE 5.x, 6.x e 7.x podem coexistir com o NetBeans IDE 8.1 no mesmo sistema. Você não precisa desinstalar as versões anteriores para instalar ou executar o NetBeans IDE 8.1.

Se você já tiver um NetBeans IDE instalado, terá a opção de importar ou não as definições de usuário do diretório existente ao executar o NetBeans 8.1 pela primeira vez.

Observação: upgrades incrementais de versão do IDE não podem ser executados neste momento. Para fazer upgrade para a versão mais recente, é preciso fazer download do instalador mais recente.

Diagnóstico e Solução de Problemas

Sobre o Diretório de Trabalho do Instalador

O instalador armazena dados sobre as sessões de instalação no diretório de trabalho do instalador. Exemplos desses dados são a lista de componentes instalados no sistema, as mensagens do log de instalação e os dados de registro.

O local default do diretório de trabalho do instalador é %USERPROFILE%\.nbi no Windows e ~/.nbi em outras plataformas. O subdiretório de log contém os arquivos de log de instalação.

Limpeza após Instalações Incompletas

Após instalações incompletas ou malsucedidas, talvez seja necessário remover as instalações anteriores do sistema.

Para Microsoft Windows e Linux:

  1. Delete o diretório de trabalho do instalador.
  2. Delete os diretórios de instalação do IDE e do servidor de aplicações GlassFish.
  3. Na plataforma Windows, delete todas as entradas do registro menos a nbi* de todas as chaves de registro: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Para OS X:

  • Arraste as pastas ou arquivos de programas para a Lixeira.

Desativando Alguns Produtos de Software Antes da Instalação

É melhor desativar o firewall, o antivírus e as aplicações de monitoramento antes de iniciar a instalação. Algumas dessas aplicações podem impedir que componentes do IDE sejam configurados corretamente durante o processo de instalação.

Por exemplo, se você tiver a aplicação Logitech Process Monitor (LVPrcSrv.exe) no sistema, desative-a antes da instalação.
