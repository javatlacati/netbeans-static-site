Ativando o C/C++/Fortran no IDE

O NetBeans IDE é um ambiente de desenvolvimento integrado modular e dinâmico, o que significa que ele pode ser alterado por meio da adição e da remoção de módulos do programa. Você pode acrescentar funcionalidades instalando módulos de plug-in ou pode remover funcionalidades desinstalado esses módulos. Consulte a ajuda on-line do NetBeans IDE para obter mais informações sobre plug-ins.

Se quiser trabalhar com os programas C, C++ ou Fortran no NetBeans IDE, a instalação do NetBeans deve incluir o plug-in do C/C++.

Embora seja denominado C/C++, o plug-in também inclui suporte para programação em Fortran.

Se ainda não tiver o NetBeans IDE 7.4 instalado

Se você ainda não tiver o NetBeans IDE instalado, faça download do pacote C/C++ ou do pacote Todos na Página de Download do NetBeans IDE 7.4. Caso não pretenda usar outras linguagens como Java e Ruby, faça download do pacote C/C++.

Se você tiver o NetBeans IDE 7.4, mas não souber se tem o plug-in do C/C++

Se já tiver o NetBeans IDE, certifique-se de que seu NetBeans IDE inclui o plug-in do C/C++ selecionando Arquivo > Novo Projeto. Se o C/C++ estiver listado como uma das Categorias, isso significa que você tem o módulo de plug-in do C/C++. Você deverá ir para a seção Instalando e Configurando Compiladores e Ferramentas.

Se tiver o NetBeans IDE 7.4 sem o plug-in do C/C++

Se o seu NetBeans IDE não mostrar a categoria de projeto C/C++ quando você selecionar Arquivo > Novo Projeto, complete as etapas a seguir para adicionar o módulo de plug-in do C/C++ ao IDE.

  1. Se sua rede utilizar um proxy, selecione Ferramentas > Opções > Geral no IDE. Em seguida, selecione Definições Manuais de Proxy, digite o Proxy HTTP e a Porta do Proxy e clique em OK.
  2. Escolha Ferramentas > Plug-ins.
  3. Na caixa de diálogo Plug-ins, clique na guia Plug-ins Disponíveis e vá até a categoria C/C++.
  4. Selecione a caixa de seleção C/C++ e clique em Instalar para iniciar o Instalador do NetBeans IDE.
  5. No Instalador do NetBeans IDE, clique em Próximo.
  6. Leia o contrato de licença, selecione a caixa de seleção para aceitar os termos do contrato e clique em Próximo.
  7. Clique em Instalar.
  8. Após a conclusão da instalação, selecione Reiniciar o IDE Agora ou Reiniciar o IDE Mais Tarde e clique em Finalizar.

Instalando e Configurando os Compiladores e as Ferramentas

O módulo NetBeans C/C++ requer um compilador C, um compilador C++, um utilitário make e um depurador gdb. Consulte as instruções a seguir para a plataforma do seu sistema de desenvolvimento.

-  Microsoft Windows
-  Oracle Solaris
-  Linux
-  Macintosh OS X

Microsoft Windows

O módulo NetBeans C/C++ foi testado com os compiladores do Cygwin e MinGW. Se você instalar o Cygwin e o MinGW, tenha cuidado ao manter seus locais de instalação separados e não misture as ferramentas do Cygwin e do MinGW em uma coleção de ferramentas no IDE.

Observação. Se você quiser utilizar o Qt com o Windows, deverá usar o MinGW. Consulte Trabalhando com Aplicações do Qt para obter instruções sobre a instalação do software MinGW e Qt.

-  Ferramentas e Compiladores do Cygwin
-  Ferramentas e Compiladores do MinGW

Ferramentas e Compiladores do Cygwin

O módulo NetBeans C/C++ foi testado com os compiladores e ferramentas do Cygwin.com mostrados a seguir.

Software ou Recurso Versão Testada Descrição
cygwin1.dll 1.7.7, 1.7.9 Ambiente Cygwin para Windows - semelhante ao Linux
gcc 4.3.4 Compilador C de Cygwin
g++ 4.3.4 Compilador C++ de Cygwin
gdb 6.8 Depurador GNU de Cygwin
make 3.81 Utilitário make de Cygwin

Se você já tiver os compiladores gcc e g++ do Cygwin, o GNU make e o depurador gdb instalados no sistema Windows e o caminho estiver configurado corretamente para encontrá-los, certifique-se de ter as versões corretas.

Para verificar as versões dos compiladores e das ferramentas de Cygwin:

  1. Verifique a versão do ambiente de Cygwin digitando os seguintes comandos em um prompt de comando do Windows:

    C:\> cygcheck -c cygwin
  2. Verifique as versões dos compiladores gcc e g++ de Cygwin, make, e gdb digitando os seguintes comandos em um prompt de comando do Windows: 
    C:\> gcc --version
C:\> g++ --version
C:\> make --version
C:\> gdb --version

Se você tiver as versões corretas, não serão necessárias configurações adicionais. Consulte Verificando a Instalação para confirmar se as ferramentas estão instaladas corretamente no NetBeans IDE.

Para instalar os compiladores gcc e g++ de GNU, make e o depurador gdb com base no site cygwin.com:

  1. Consulte o Guia do Usuário do Cygwin para obter informações completas sobre como instalar e utilizar o Cygwin.
  2. Faça download do programa Cygwin setup-x86.exe (instalação de 32bits) ou setup-x86_64.exe (instalação de 64 bits) clicando em Instalar Cygwin na barra de navegação esquerda, ou clicando no link direto setup-x86.exe ou setup-x86_64.exe.

    Observação: a verificação de bits do SO Windows, Java e NetBeans IDE instalado em sua máquina deve corresponder. Por exemplo, se você estiver executando a versão de 64 bits do sistema operacional Windows, o Java de 64bits e o Cygwin de 64bits (que é setup-x86_64.exe) terão instalação obrigatória. Consulte a questão 234221 para obter detalhes.

  3. Execute o instalador de Cygwin baixado. Aceite os defaults até chegar na página Selecionar Conexão de Internet. Nessa página, selecione a opção que melhor se aplica ao seu caso. Clique em Próximo.
  4. Na página Selecionar Local de Download, escolha um local de download que esteja relativamente próximo a você. Clique em Próximo.
  5. Na página Selecionar Pacotes, selecione os pacotes para download. Clique no sinal + próximo a Devel para expandir a categoria de ferramentas de desenvolvimento. Redimensione a janela para que possa ver mais de uma janela ao mesmo tempo.
  6. Selecione cada pacote que você deseja fazer download clicando no label Ignorar ao lado dele, o que revela o número da versão do pacote para download. No mínimo, selecione
    • -core gcc: compilador C
    • G++ compilador C ++-gcc:
    • gdb: O Depurador GNU
    • make: a versão do GNU do utilitário 'make'
    Os pacotes exigidos pelos pacotes que você selecionar também serão automaticamente selecionados.
  7. Clique em Próximo para estabelecer conexão com o site de download e fazer download dos pacotes que selecionou. Clique em Finalizar quando a instalação estiver concluída.
  8. Agora, adicione o diretório do compilador do Cygwin ao caminho para permitir que o NetBeans IDE encontre a coleção de ferramentas:

    1. Abra o Painel de Controle:
      - No Windows XP, selecione Iniciar > Configurações > Painel de Controle e clique duas vezes em Sistema.
      - No Windows 7, digite var na caixa de pesquisa do menu Iniciar para localizar rapidamente um link para editar as variáveis de ambiente do sistema.
    2. Selecione a guia Avançado e clique em Variáveis de Ambiente.
    3. No painel Variáveis do Sistema da caixa de diálogo Variáveis de Ambiente, selecione a variável Path e clique em Editar.
    4. Adicione o caminho ao diretório cygwin-directory\bin à variável Path e clique em OK. Por default, cygwin-directory é C:\cygwin (para distribuição do Cygwin de 32 bits) ou C:\cygwin64 (para distribuição do Cygwin de 64 bits). Os nomes do diretório devem ser separados por ponto-e-vírgula. O caminho editado deve ser semelhante a %SystemRoot%\system32;%SystemRoot%;C:\Program Files\QuickTime\QTSystem;C:\cygwin\bin
    5. Clique em OK na caixa de diálogo Variáveis de Ambiente e na caixa de diálogo Propriedades do Sistema.
    6. Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Ferramentas e Compiladores do MinGW

O NetBeans IDE 7.4 foi testado com o MinGW (Minimalist GNU para Windows) e com o ambiente MSYS (Minimal System), que é semelhante ao Unix. As versões testadas e as instruções de instalação são mostradas abaixo.

Observação. Se você quiser usar o Qt com Windows, consulte o artigo Trabalhando com Aplicações do Qt e obtenha instruções sobre a instalação do software MinGW e Qt.

Software ou Recurso Versão Testada Descrição
gcc 4.6.2 Compilador C
g++ 4.6.2 Compilador C++ de MinGW
gdb 7.4 Depurador GNU de MinGW
make 3.81 Utilitário make de MSYS
Observe que o make de MinGW não é suportado

Para instalar os compiladores GNU, make e o depurador gdb com base no site mingw.com:

  1. Efetue log-in no Windows utilizando uma conta com privilégios de administrador do computador.
  2. Efetue download do instalador do MinGW em http://sourceforge.net/projects/mingw.
  3. Execute o instalador do MinGW, que deve ter um nome de arquivo semelhante a mingw-get-inst-20120426.exe.
  4. Aceite o C:\MinGW default como pasta de destino, se possível, para diminuir qualquer possível dificuldade no uso de compiladores provenientes de outro local.
  5. No instalador do MinGW, selecione os seguintes componentes para instalação:
    • C
    • C++
    • Fortran (se você for trabalhar em programas Fortran)
    • Sistema Básico MSYS
    • Kit de Ferramentas para Desenvolvedor do MinGW
  6. Clique em Instalar e aguarde alguns minutos enquanto o programa de instalação faz o download dos componentes que você selecionou.
  7. Clique em Finalizar quando o download dos pacotes for concluído.

Você deve adicionar os caminhos até os binários das ferramentas MinGW e MSYS ao seu PATH. Se você tiver instalado no local default, os caminhos serão C:\MinGW\bin e C:\MinGW\MSYS\1.0\bin.

Para editar sua variável de ambiente PATH no Windows:

  1. Abra a janela Variáveis de Ambiente:
    • No Windows XP e Windows 2000: clique com o botão direito do mouse em Meu Computador > Propriedades > guia Avançadas > botão Variáveis do Ambiente.
    • No Windows Vista e Windows 7: clique com o botão direito do mouse em Meu Computador > Propriedades > link Configurações Avançadas do Sistema > botão Variáveis do Ambiente.
  2. Na janela Variáveis do Ambiente, selecione a variável Path na seção Variáveis do Sistema e clique em Editar.
  3. No final do caminho, insira um ponto-e-vírgula e adicione os caminhos até os executáveis para MinGW e MSYS. Use pontos-e-vírgulas entre os caminhos e não use nenhum espaço. Tenha cuidado para não remover nada do PATH ou seu computador pode não funcionar corretamente.

    Quando terminar, o caminho deve ser semelhante ao seguinte:

    %SystemRoot%\system32;%SystemRoot%;%SystemRoot%\System32\Wbem;
%SYSTEMROOT%\System32\WindowsPowerShell\v1.0;C:\MinGW\bin;C:\MinGW\MSYS\1.0\bin
  4. Clique em OK na janela Variáveis do Ambiente.

Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Oracle Solaris

Nas plataformas Oracle Solaris, é possível utilizar as ferramentas do GNU ou as ferramentas do Oracle Solaris Studio. As ferramentas do GNU estão incluídas no Oracle Solaris 11 no diretório /usr/sfw/bin por default. O software Oracle Solaris Studio é um conjunto de ferramentas para desenvolvedores cujo download é gratuito. Os compiladores do Oracle Solaris Studio foram otimizados para o hardware do Oracle Sun e facilitam a produção de binários Oracle Solaris com ajuste de desempenho.

O NetBeans IDE 7.4 foi testado com os compiladores e ferramentas a seguir.

Compiladores Oracle Solaris Studio 12.3

Caso queira usar os compiladores do Oracle Solaris Studio 12.3 no Oracle Solaris 11:

  • Se você tiver o software Oracle Solaris Studio 12.3 instalado, certifique-se de que o /diretório de instalaçãosolarisstudio12.3/bin esteja no caminho antes de iniciar o NetBeans IDE. A localização default no Solaris 11 é /opt/solarisstudio12.3/bin
  • Caso você não tenha o software Oracle Solaris Studio 12.3 instalado, poderá obtê-lo por download de graça em http://www.oracle.com/technetwork/server-storage/solarisstudio/downloads/index-jsp-141149.html.

Para fazer download e instalar os compiladores do Oracle Solaris Studio 12.3 no Oracle Solaris 11:

  1. Crie um diretório para o arquivo submetido a download. Você deverá ter permissão de gravação neste diretório.
  2. Faça download do arquivo instalador do pacote para sua plataforma no diretório de download.
  3. Vá ao diretório de download e descompacte o arquivo submetido a download. 
    bzcat filename | /bin/tar xvf -
  4. Siga as instruções dispostas no Capítulo 2 do Guia de Instalação do Oracle Solaris Studio 12.3 para instalar o compilador C++, o compilador C++ e os patches necessários do Solaris.
  5. Edite seu PATH para adicionar o caminho ao software Oracle Solaris Studio 12.3 antes de iniciar o NetBeans IDE.
  6. Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Para fazer download e instalar os compiladores do Oracle Solaris Studio 12.3 no Oracle Solaris 11:

  • Vá até a página de download do Oracle Solaris Studio e siga as instruções para Oracle Solaris 11 na coluna de Instalação de Pacotes.

Compiladores GNU e GNU make

Se quiser utilizar os compiladores GNU e GNU make:

  • Se você tiver uma instalação padrão do Oracle Solaris 11, os compiladores e gmake estarão instalados em /usr/sfw/bin. Certifique-se de que este local esteja no PATH antes de iniciar o NetBeans IDE.
  • Se os compiladores e o gmake não estiverem instalados no seu sistema, você poderá fazer download em http://pkg.oracle.com.

Para fazer download e instalar os compiladores GNU e o utilitário make:

  1. Faça download e instale os pacotes com o comando pkg install digitando pkg install gcc-3 gnu-make no prompt de comando.
  2. Certifique-se de incluir o diretório do compilador GNU e GNU make no caminho antes de iniciar o NetBeans IDE.
  3. Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Depurador gdb

Se você utilizar os compiladores do Oracle Solaris Studios e o Solaris make, ou os compiladores GNU e o GNU make,você deverá ter o depurador gbd para depurar aplicações no NetBeans IDE. É possível fazer download do gdb do site http://pkg.oracle.com.

Para fazer download e instalar o gdb para Oracle Solaris 11:

  1. Faça download e instale o pacote com o comando pkg install digitando pkg install gdb no prompt de comando.
  2. Certifique-se de incluir o caminho para gdb no PATH antes de iniciar o NetBeans IDE.
  3. Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Compiladores Oracle Solaris Studio 12.2 no Linux

Se quiser usar os compiladores do Oracle Solaris Studio 12.3 em um Sistema Operacional Linux:

  • Se você tiver o software Oracle Solaris Studio 12.3 instalado, certifique-se de que o /diretório de instalação/solarisstudio12.3/bin esteja no caminho antes de iniciar o NetBeans IDE. O local default é /opt/oracle/solarisstudio12.3/bin durante a instalação com pacotes do Linux.
  • Caso você não tenha o software Oracle Solaris Studio 12.3 instalado, poderá obtê-lo por download de graça em http://www.oracle.com/technetwork/server-storage/solarisstudio/downloads/index-jsp-141149.html.

Para fazer download e instalar os compiladores do Oracle Solaris Studio 12.3:

  1. Crie um diretório para o arquivo submetido a download. Você deverá ter permissão de gravação neste diretório.
  2. Se o seu browser estiver configurado para fazer download em um local específico como na Área de Trabalho ou em um diretório de Downloads sem solicitar permissão, defina as preferências do browser para que o download seja feito no diretório que você criou. No Firefox, o diretório de download é definido em Editar > Preferências > Principal.
  3. Faça download do arquivo para sua plataforma no diretório de download que você criou. Se estiver usando o Ubuntu, faça download do arquivo tar de instalação no diretório onde deseja instalá-lo, pois os pacotes são para o Oracle Linux, Red Hat Enterprise Linux e o SuSE Linux Enterprise System. Observe também que o Ubuntu não é uma plataforma oficialmente suportada para o Oracle Solaris Studio IDE, mas os compiladores foram testados para serem utilizados no NetBeans IDE e não apresentaram problemas.
  4. Vá ao diretório de download e descompacte o arquivo submetido a download. 
    bzcat filename | tar xvf -
  5. Se você tiver feito download dos pacotes SUSE ou RPM, siga as instruções dispostas no Capítulo 2 do Guia de Instalação do Oracle Solaris Studio 12.3 para instalar o compilador C, o compilador C++ e o compilador Fortran.
  6. Edite seu PATH para adicionar o caminho ao software Oracle Solaris Studio antes de iniciar o NetBeans IDE.
  7. Consulte Verificando a Instalação para confirmar se as ferramentas foram instaladas corretamente para o NetBeans IDE.

Para fazer download e instalar o depurador do GNU no Oracle Linux ou no Red Hat Enterprise Linux, digite:

yum install gdb

Para fazer download e instalar o depurador GNU em Ubuntu, digite:

sudo apt-get install gdb

Verificando a Instalação

Para verificar se a instalação está correta, inicie o NetBeans IDE, crie um projeto de amostra e execute-o no depurador gdb.

Inicie o NetBeans IDE

Para iniciar o IDE, adote um dos seguintes procedimentos:

  • Clique duas vezes no ícone do NetBeans IDE na área de trabalho.
  • Nas plataformas Linux ou Solaris, acesse o subdiretório bin da instalação do NetBeans e digite ./netbeans.
  • Inicie o NetBeans IDE por meio do menu na área de trabalho.
    Esse método está disponível nas plataformas Windows, mas talvez não esteja disponível no Oracle Solaris 11 e em algumas plataformas Linux.

Crie um Projeto de Amostra para Testar as Ferramentas

Em todas as plataformas, é possível criar um projeto de amostra para testar a instalação do compilador e executá-lo no depurador para testar a instalação do gdb.

Para criar um projeto de amostra e executá-lo no depurador:

  1. Abra o assistente para Novo Projeto em Arquivo > Novo Projeto.
  2. No painel Categorias na página Escolher Projeto do assistente, expanda a categoria Amostras e selecione a subcategoria C/C++.
  3. No painel Projetos, selecione o projeto Welcome. Clique em Próximo.
  4. Na página Nome e Local do Projeto, clique em Finalizar.
  5. Na janela Projetos, clique com o botão direito do mouse no projeto Welcome_1 e escolha Construir. Se os compiladores e o utilitário make estiverem instalados corretamente e o caminho que leva a eles tiver sido definido, a saída da construção do projeto será exibida na janela Saída e a construção do projeto será bem-sucedida.
  6. Expanda o nó Arquivos de Código-Fonte do projeto e clique duas vezes no arquivo welcome.cc para abri-lo no Editor de Código-Fonte.
  7. Defina um ponto de interrupção clicando na margem esquerda da janela Editor de Código-Fonte ao lado de qualquer linha.
  8. Clique com o botão direito do mouse no projeto e escolha Depurar. Se o depurador gdb estiver instalado corretamente e o caminho para ele estiver definido, o gdb será iniciado, as guias do Depurador serão exibidas e a aplicação Welcome será executada e interrompida no ponto de interrupção.
  9. Escolha Depurar > Continuar para executar a aplicação até o final.
  10. Se o projeto não for construído ou o depurador não funcionar, consulte a seção a seguir.

Problemas com as Ferramentas de Solução de Problemas

  1. Selecione Ferramentas > Opções e clique em C/C++ no painel superior da caixa de diálogo Opções.
  2. Na guia Ferramentas de Construção, examine a lista Coleção de Ferramentas para ver se sua coleção de ferramentas está listada.
  3. Selecione a coleção de ferramentas se ela estiver listada e verifique os caminhos para as ferramentas. Se o caminho do Diretório Base estiver incorreto, talvez você tenha cometido algum erro ao definir sua variável de ambiente Path. Consulte de novo as instruções para definir o caminho na seção para sua plataforma neste documento e faça a correção do caminho, se necessário.
  4. Se a coleção de ferramentas não estiver listada, clique em Restaurar Defaults. Isso fará com que o IDE execute uma nova varredura no seu ambiente para procurar por ferramentas, e a coleção de ferramentas deverá ser encontrada se a variável de ambiente Path estiver correta.
  5. Se ainda tiver um problema, tente adicionar uma nova coleção de ferramentas e especifique o caminho para as ferramentas, da seguinte forma:
    1. Clique em Adicionar abaixo da lista Coleção de Ferramentas.
    2. Procure pelo diretório que contém os executáveis para os compiladores, depuradores e para o utilitário make. Normalmente, trata-se do diretório bin.
    3. Selecione o diretório e clique em Abrir. O IDE deverá ser capaz de determinar a família das ferramentas e atualizar os outros campos na caixa de diálogo de forma apropriada. Caso contrário, um erro será exibido.
    4. Se as ferramentas forem corretamente identificadas, clique em OK na caixa de diálogo Adicionar Coleção de Ferramentas. A nova coleção deverá estar exibida na lista.
    5. Selecione a nova coleção de ferramentas e clique em Default para assegurar que esta coleção de ferramentas seja automaticamente usada para novos projetos.

    Se não conseguir solucionar o problema, solicite ajuda ao Fórum de Usuários do NetBeans C/C++.
