Требуемое программное обеспечение

Для установки среды IDE NetBeans требуется комплект разработчика Java (JDK) 7 (обновление 10) или более поздняя версия. Последнее обновление JDK 7 можно загрузить по адресу: http://www.oracle.com/technetwork/java/javase/downloads.

Для установки пакетов PHP и C/C++ NetBeans требуется только установленная и функционирующая среда выполнения Java (JRE) 7. Если планируется использование любых функциональных возможностей Java, требуется JDK 7.

Чтобы использовать функции JavaFX 2.2 (или более поздней версии) в IDE NetBeans 7.4, требуется использовать JDK 7 (обновление 6) или более поздней версии.

Примечание.

  • JavaFX 2.2 входит в комплект последней версии JDK 7 для Windows, Mac и Linux.
  • Чтобы использовать поддержку JavaFX в IDE, для NetBeans IDE 7.4, в отличие от предыдущих версий NetBeans IDE, не требуется настройка платформы Java с поддержкой FX . Разрабатывать проекты JavaFX в IDE можно, если установлена любая стандартная платформа Java с JDK 7 (обновление 6) или более поздней версии.
  • Чтобы использовать функции JavaFX в IDE NetBeans, рекомендуется обновить JDK до версии JDK 7 (обновление 10) или более поздней версии.

В начало

Варианты загрузки программы установки

На странице загрузок IDE NetBeans можно получить одну из нескольких программ установки, каждая из которых содержит базовую среду IDE и дополнительные средства.

Доступны следующие программы установки:

  • Java SE. Поддерживает все стандартные функциональные возможности разработки Java SE, а также поддержку разработки платформы NetBeans и JavaFX 2.2 SDK (или более поздней версии).
  • Java EE. Предоставляет средства для разработки приложений Java SE и Java EE, а также поддержку разработки платформы NetBeans и JavaFX 2.2 SDK (или более поздней версии). В состав данного варианта загрузки входит также сервер GlassFish Server Open Source Edition 4.0 и программный пакет Apache Tomcat 7.0.41.
  • C/C++. Реализована поддержка разработки на языках C, C++, Qt, Фортран и на языке ассемблера.
  • PHP. Предоставляет средства для разработки PHP 5.x и поддержки платформ Zend и Symfony.
  • Все. Это вариант полной загрузки, который содержит все доступные среды выполнения и технологии для среды IDE NetBeans.

В начало

Индивидуализация процесса установки

Вариант загрузкиВсе позволяет выбрать средства и среды выполнения для установки. Щелкните ссылку "Настроить" на странице "Добро пожаловать" и выберите требуемые компоненты. Дополнительные сведения см. в разделе Установка программного обеспечения.

Для добавления функциональных возможностей к среде IDE NetBeans используется диспетчер подключаемых модулей NetBeans. Допустим, например, что вы начинаете с загрузки и установки пакета PHP. Затем вы приняли решение испытать функциональные возможности C/C++. Для этого в среде IDE перейдите к диспетчеру подключаемых модулей NetBeans (выберите "Сервис > Подключаемые модули") и добавьте к существующей установке пакет "C/C++".

Ниже приводится ряд важных примечаний к процессу установки и настройки некоторых компонентов среды IDE NetBeans:

  • Сервер GlassFish Server Open Source Edition 4.0 включен в варианты загрузки Java EE и Все, но не устанавливается по умолчанию ни в одном из этих вариантов. Для установки Apache Tomcat выберите нужный вариант из перечисленных ниже:
    • из загрузки Java EE запустите программу установки и установите флажок GlassFish Server Open Source Edition 4.0 на странице приветствия;
    • из загрузки Все запустите программу установки и выберите GlassFish Server Open Source Edition 4.0 в диалоговом окне 'Настроить устанавливаемые компоненты'.
    После установки сервера GlassFish необходимо изменить пароль администратора по умолчанию. Подробные сведения см. в разделе FaqSettingGlassFishPassword.
  • Пакет Apache Tomcat 7.0.41 включен в варианты загрузки Java EE и Все, но не устанавливается по умолчанию ни в одном из этих вариантов. Для установки Apache Tomcat выберите нужный вариант из перечисленных ниже:
    • из загрузки Java EE запустите программу установки и установите флажок Apache Tomcat 7.0.41 на странице приветствия;
    • из загрузки Все запустите программу установки и выберите Apache Tomcat 7.0.41 в диалоговом окне "Настроить устанавливаемые компоненты".
  • При разработке проектов на HTML5 и JavaScript для мобильных устройств (Android и iOS) требуется дополнительно установить комплекты SDK и другие средства разработки для целевой платформы разработки. Дополнительные сведения см. на wiki-странице Мобильные браузеры.
  • Для OS X настройку установки среды IDE NetBeans можно выполнить из загрузок Все и Java EE. В случае установки среды IDE из загрузок Java SE, C/C++ и PHP индивидуальная настройка невозможна.
  • Также можно загрузить не зависящий от платформы zip-файл и запустить исполняемый файл среды IDE NetBeans. Обратите внимание, что GlassFish Server Open Source Edition и Apache Tomcat доступны только в составе программ установки для определенных платформ.
  • Чтобы сделать шаблоны демонстрационного проекта недоступными при установке среды IDE NetBeans, воспользуйтесь информацией на страницеСкрытие демонстрационных проектов: ЧАВО.

В начало

Начало загрузки

  1. Перейдите по ссылке https://netbeans.org/downloads/
  2. Выберите соответствующую платформу и язык из выпадающего списка в верхней правой части страницы. Также для загрузки и использования можно выбрать zip-файл, не зависящий от платформы.
  3. Нажмите кнопку "Загрузить", соответствующую выбранному варианту загрузки.
  4. Сохраните файл программы установки на вашем компьютере.

В начало

Установка программного обеспечения

Для установки среды IDE следуйте указаниям, приведенным в этом разделе. Данные инструкции по установке применяются ко всем поддерживаемым платформам. Список поддерживаемых платформ и системных требований приведен в примечаниях к выпуску.

ОС Microsoft Windows, Solaris и Linux

Для установки программного обеспечения:

  1. После завершения загрузки запустите программу установки.
    • Для Windows исполняемый файл программы установки имеет расширение .exe. Чтобы запустить файл программы установки, дважды щелкните его.
    • Для платформ Solaris и Linux файл программы установки имеет расширение .sh. На этих платформах необходимо выполнить следующую команду для исполняемых файлов программы установки: chmod +x <installer-file-name>
  2. В случае загрузки пакета Все или Java EE возможна пользовательская настройка установки. Выполните следующие действия на странице приветствия мастера установки:
    1. Нажмите кнопку "Настроить".
    2. В диалоговом окне "Изменить устанавливаемые компоненты" выберите нужные варианты.
    3. Нажмите кнопку "ОК".
  3. На странице "Добро пожаловать" мастера установки нажмите кнопку "Далее".
  4. На странице "Лицензионное соглашение" ознакомьтесь с лицензионным соглашением, установите флажок принятия условий и нажмите кнопку "Далее".
  5. На странице "Лицензионное соглашение JUnit" определите необходимость установки JUnit, щелкните соответствующий параметр и нажмите кнопку "Далее".
  6. На странице установки среды IDE NetBeans выполните следующие действия:
    1. Выберите каталог для установки среды IDE NetBeans по умолчанию или укажите другой каталог.

      Примечание. Каталог установки должен быть пустым, а профиль пользователя, который используется программой установки, должен предусматривать права на чтение/запись для этого каталога.

    2. Выберите установку комплекта JDK для использования со средой IDE NetBeans по умолчанию или выберите другую установку в раскрывающемся списке. Если мастеру установки не удается обнаружить совместимую установку комплекта JDK для использования со средой IDE NetBeans, это означает, что комплект JDK не установлен в местоположении по умолчанию. В этом случае укажите путь к установленному пакету JDK и нажмите кнопку "Далее" или отмените текущую установку. После установки необходимой версии пакета JDK можно перезапустить установку.

      Примечание. Если версия JDK является более старой, чем рекомендованная версия JDK 7 (обновление 10), загрузите и установите последнее обновление JDK со страницы Загрузки Java SE и перезапустите программу установки IDE NetBeans.

  7. При появлении страницы установки GlassFish Server Open Source Edition 4.0 выберите каталог установки по умолчанию или укажите иное местоположение для установки.
  8. Если выполняется установка сервера приложений Apache Tomcat, выберите на его странице установки каталог установки по умолчанию или укажите другое местоположение для установки. Нажмите кнопку "Далее".
  9. На странице "Сводка" выполните следующие действия:
    1. Проверьте список выбранных для установки компонентов.
    2. Установите флажок "Проверять наличие обновлений", если вы хотите проверять наличие доступных обновлений с помощью центра обновления; при этом в процессе установки будет установлена библиотека JUnit (при условии принятия лицензионного соглашения на шаге 5.)
    3. Убедитесь, что на диске достаточно свободного места.
  10. Для запуска процесса установки нажмите кнопку "Установить".
  11. На странице "Установка завершена" по желанию укажите анонимные данные об использовании и нажмите кнопку "Далее".

Примечание. В случае возникновения проблем при установке программного обеспечения обратитесь к разделу Устранение проблем, в котором содержатся инструкции по устранению проблем и временные решения для неустранимых проблем, с которыми пользователи могут столкнуться в процессе установки.

OS X

Для установки программного обеспечения:

  1. После завершения загрузки запустите программу установки. Файл программы установки имеет расширение .dmg.
  2. На открывшейся панели дважды щелкните значок пакета. Пакет имеет расширение .mpkg. Запускается мастер установки.
  3. В отобразившемся диалоговом окне "Пакет выполнит запуск программы для проверки возможности установки программного обеспечения" нажмите кнопку "Продолжить".
  4. На странице "Введение" мастера установки нажмите кнопку "Продолжить".

    Примечание. Если версия JDK является более старой, чем рекомендованная версия JDK 7 (обновление 10), загрузите и установите последнее обновление JDK со страницы Загрузки Java SE и перезапустите программу установки IDE NetBeans.

  5. Прочтите лицензионное соглашение и нажмите кнопку "Продолжить". Для принятия условий лицензионного соглашения нажмите кнопку "Принимаю" во всплывающем окне.
  6. На странице "Выбор назначения" выберите диск и нажмите кнопку "Продолжить".
  7. В случае загрузки пакета Все или Java EE возможна пользовательская настройка установки. В нижней левой части последней панели мастера установки нажмите кнопку "Настроить". Отображается дерево программ.
  8. Выберите программы, которые требуется установить.
  9. Введите имя администратора и пароль и нажмите кнопку "ОК" для запуска установки.

Пакет, не зависящий от платформы

Для запуска программного обеспечения:

  1. После загрузки не зависящего от платформы zip-файла распакуйте его в любую папку на вашем компьютере.
  2. Запустите исполняемый файл, расположенный в каталоге netbeans/bin.
  3. Примите условия лицензионного соглашения.
  4. На странице "Лицензионное соглашение JUnit" определите необходимость установки JUnit, щелкните соответствующий параметр и нажмите кнопку "Далее". Начнется установка среды IDE NetBeans.

Примечание. Если совместимый установленный комплект JDK не обнаружен, может потребоваться вручную добавить путь к этому пакету в папку установки JDK. Выполните следующие действия:

  1. Откройте в текстовом редакторе файл netbeans.conf, расположенный в каталоге netbeans/etc.
  2. Укажите местоположение совместимого установленного комплекта JDK в качестве значения параметра netbeans_jdkhome. В ОС Windows установка по умолчанию выполняется в местоположение C:\Program Files\Java\jdk1.7.0_10 (или подобное).
  3. Сохраните файл netbeans.conf и запустите исполняемый файл в каталоге netbeans/bin.

Примечание. GlassFish Server Open Source Edition и Apache Tomcat доступны только в составе программ установки для определенных платформ.

Примечание. Для использования уже установленных компонентов GlassFish Server Open Source Edition и/или Apache Tomcat в среде IDE NetBeans выберите Серверы в меню Сервис. В результате отображается мастер добавления экземпляра сервера, который поможет зарегистрировать экземпляр нужного веб-сервера в среде IDE NetBeans.

Удаление программного обеспечения

Для удаления среды IDE следуйте инструкциям, приведенным в этом разделе. Данные инструкции по удалению применяются ко всем поддерживаемым платформам. Список поддерживаемых платформ приведен в примечаниях к выпуску.

Microsoft Windows

При необходимости можно удалить среду IDE NetBeans, GlassFish Server Open Source Edition 4.0 и Apache Tomcat, применив следующие процедуры.

Для удаления среды IDE:

  1. Завершите работу среды IDE.
  2. На панели управления выберите 'Установка и удаление программ', а затем выберите IDE NetBeans 7.4 и щелкните 'Удалить'.
  3. Для удаления компонентов GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и установите соответствующие флажки.
  4. Установите флажок для каталога пользователя NetBeans, если требуется удалить этот каталог вместе со средой IDE, и нажмите "Удалить".
  5. На странице "Сводка" нажмите кнопку "Удалить".
  6. После завершения удаления нажмите кнопку "Готово".

Удаление компонентов GlassFish Server Open Source Edition 4.0:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите GlassFish Server Open Source Edition 4.0 и нажмите "Удалить".

Для удаления компонентов Apache Tomcat:

  1. Завершите работу среды IDE.
  2. На панели управления выберите "Установка и удаление программ", затем выберите Apache Tomcat 7.0.41 и нажмите "Удалить".

ОС Solaris и Linux

При необходимости можно удалить среду IDE NetBeans, GlassFish Server Open Source Edition 4.0 и Apache Tomcat, применив следующие процедуры.

Удаление среды IDE:

  1. Завершите работу среды IDE.
  2. В каталоге установки IDE запустите файл uninstall.sh.
  3. Для удаления GlassFish и Tomcat вместе со средой IDE на странице "Сводка" проверьте правильность версий и установите соответствующие флажки.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Удаление компонентов GlassFish Server Open Source Edition 4.0:

  1. Завершите работу среды IDE.
  2. Укажите каталог установки сервера приложений.

    Примечание. Местоположение установки по умолчанию зависит от типа установки и платформы. В следующей таблице приводятся каталоги установки сервера приложений по умолчанию для различных платформ.

    Платформа Установка от имени... Установка GlassFish 4.0
    Windows пользователь с правами администратора %ProgramFiles%\glassfish-4.0
    Windows пользователь %USERPROFILE%\glassfish-4.0
    ОС Solaris корень /opt/glassfish-4.0
    ОС Solaris, Linux пользователь ~/glassfish-4.0
    Linux корень /usr/local/glassfish-4.0
  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

Удаление пакета Apache Tomcat:

  1. Завершите работу среды IDE.
  2. Укажите каталог установки Apache Tomcat.

    Примечание. Местоположение установки по умолчанию зависит от типа установки и платформы. В следующей таблице приводятся каталоги установки сервера приложений по умолчанию для разных платформ.

    Платформа Установка от имени... Установка Apache Tomcat 7.0.41
    Windows пользователь с правами администратора %ProgramFiles%\Apache Software Foundation\Apache Tomcat 7.0.41
    Windows пользователь %USERPROFILE%\Apache Software Foundation\Apache Tomcat 7.0.41
    ОС Solaris корень /opt/apache-tomcat-7.0.41
    ОС Solaris, Linux пользователь ~/apache-tomcat-7.0.41
    Linux корень /usr/local/apache-tomcat-7.0.41
  3. В каталоге установки сервера приложений запустите файл uninstall.sh.
  4. На странице "Сводка" нажмите кнопку "Удалить".
  5. После завершения удаления нажмите кнопку "Готово".

OS X

Для удаления среды IDE и сервера приложений:

  1. Завершите работу программного обеспечения.
  2. Перетащите папки или файлы программы в корзину. Обратите внимание, что местоположением для среды IDE и сервера приложений по умолчанию является /Applications/NetBeans/.

Поддержка обновлений и множественных установок

Несколько установок среды IDE NetBeans 5.x, 6.x и 7.x могут существовать совместно со средой IDE NetBeans 7.4 в одной и той же системе. Для установки или запуска NetBeans IDE 7.4 не требуется удалять более ранние версии.

Если на вашем компьютере уже установлена среда IDE NetBeans, при желании вы можете импортировать настройки пользователей из существующего каталога пользователей при первом запуске IDE NetBeans 7.4.

Примечание. На данном этапе невозможно выполнить последовательные обновления версий среды IDE. Для обновления до новейшей версии необходимо загрузить последнюю версию программы установки.

Устранение проблем

О рабочем каталоге программы установки

Программа установки сохраняет данные о сеансах установки в рабочем каталоге. Эти данные включают список установленных в системе компонентов, сообщения журнала установки и данные из реестра.

Местоположением рабочего каталога программы установки по умолчанию является %USERPROFILE%\.nbi для ОС Windows и ~/.nbi для других платформ. Файлы журнала установки находятся в подкаталоге журнала.

Очистка после незавершенных установок

После незавершенных или неуспешных установок может потребоваться очистка системы.

Для ОС Microsoft Windows, Solaris и Linux:

  1. Удалите рабочий каталог программы установки.
  2. Удалите каталоги установки среды IDE и сервера приложений GlassFish.
  3. На платформе Windows следует удалить все записи вида nbi* из следующих ключей реестра: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall, HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

Для OS X:

  • Перетащите папки или файлы программы в корзину.

Отключение некоторых программ до установки

Перед началом установки рекомендуется отключить брандмауэр, антивирусное программное обеспечение и приложения мониторинга. Некоторые из этих приложений могут стать причиной некорректной настройки компонентов среды IDE в процессе установки.

Например, если в системе работает приложение мониторинга процессов Logitech (LVPrcSrv.exe), перед установкой его рекомендуется отключить.
